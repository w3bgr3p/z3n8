

Мониторинг и управление ZennoBrowser через ZB API. Показывает запущенные инстансы браузера, профили, прокси и потоки.

Путь: `?page=zb`

При открытии страница автоматически пытается подключиться к ZB API.

---

## Статус подключения

Строка под заголовком показывает текущее состояние:

| Элемент       | Описание                                                                       |
| ------------- | ------------------------------------------------------------------------------ |
| Точка         | Зелёная — подключено, красная — ошибка, серая — не проверялось                 |
| Host          | Адрес ZB API из конфига (`zbHost`)                                             |
| Key           | Статус API-ключа — `●●●●` если задан, `not set` если отсутствует               |
| **↻ Connect** | Переподключиться вручную                                                       |
| ⚙ Config      | Переход на [[shift + ctrl + 0.    Config\|Config]] для настройки ключа и хоста |

## Счётчики

Четыре карточки в шапке:

|Карточка|Описание|
|---|---|
|Running|Число запущенных инстансов браузера|
|Profiles|Общее число профилей в ZB|
|Proxies|Число прокси, настроенных в ZB|
|Threads|Число активных потоков|
|Uptime|Время с момента подключения к API|

## Instances

Вкладка по умолчанию. Список запущенных инстансов браузера.

|Колонка|Описание|
|---|---|
|Profile ID|ID профиля ZB|
|PID|ID процесса в ОС|
|WS Endpoint|WebSocket endpoint для подключения к браузеру|
|Uptime|Время жизни процесса|

**Кнопки на строке:**

|Кнопка|Действие|
|---|---|
|**Stop**|Остановить инстанс через ZB API|
|**Kill**|Принудительно завершить процесс через `Process.Kill` (минуя ZB API). Подтверждение обязательно|
|**WS**|Скопировать WS Endpoint в буфер|
|**ID**|Скопировать Profile ID в буфер|

**↻ Refresh** — обновить список вручную. Автообновление — каждые 10 секунд.

**⛔ Stop All** — остановить все запущенные инстансы (с подтверждением).

## Profiles

Список всех профилей ZennoBrowser.

|Колонка|Описание|
|---|---|
|Name|Имя профиля|
|Folder|Папка профиля в ZB|
|Tags|Теги профиля|
|Status|Текущий статус профиля|
|Last Start|Дата и время последнего запуска|

**Фильтры:**

- Поиск по имени и тегам (подстрока)
- Фильтр по папке

**Сортировка** — клик по заголовку колонки Name, Folder, Status, Last Start. Повторный клик меняет направление (↑ / ↓).

**▶ Start** — запустить инстанс для этого профиля.

**ID** — скопировать Profile ID в буфер.

## Threads

Список активных thread-токенов ZB.

|Колонка|Описание|

## Edge cases

**Точка красная после Connect** — ZB API недоступен. Проверь что ZennoBrowser запущен и доступен по адресу из `Config → ZB Host`. Убедись что ZB API Key задан в [[shift + ctrl + 0.    Config\|Config]] .

**Host показывает `localhost:8160`** — значение по умолчанию, `zbHost` не задан в конфиге. Задай адрес в [[shift + ctrl + 0.    Config\|Config]] .

**Kill вместо Stop** — используй Kill только если Stop не работает (инстанс завис). Kill обходит ZB API и завершает процесс напрямую — ZB может не освободить ресурсы корректно.

**Profiles пустые после подключения** — профили загружаются отдельно от инстансов. Перейди на вкладку Profiles — данные загрузятся при первом открытии вкладки.
