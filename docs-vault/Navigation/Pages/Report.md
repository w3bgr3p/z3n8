

Сводный отчет по проектам и социальным сетям. Heatmap выполнения задач по аккаунтам, статистика соцсетей, мониторинг процессов.

Путь: `/report.html`

---

## Обзор

Union Report — централизованная панель мониторинга для отслеживания выполнения daily-проектов и статуса аккаунтов в социальных сетях. Отображает данные в виде heatmap с цветовой индикацией давности и статуса выполнения.

Автообновление каждые 60 секунд.

---

## Header

**📊 Union Report** — заголовок с датой/временем генерации и user ID.

Формат: `📊 Union Report DD.MM.YYYY HH:MM:SS id: {userId}`

## Social Networks Section

### Summary Cards

Верхние карточки со статистикой:

|Карточка|Описание|
|---|---|
|**TOTAL ACCOUNTS**|Общее количество отслеживаемых аккаунтов|
|**TWITTER**|Количество Twitter-аккаунтов / активных|
|**GITHUB**|Количество GitHub-аккаунтов / активных|
|**DISCORD**|Количество Discord-аккаунтов / активных|
|**TELEGRAM**|Количество Telegram-аккаунтов / активных|

### Social Networks HeatMap

Сетка из ячеек, каждая ячейка = 1 аккаунт, разделена на 4 квадрата (Twitter, GitHub, Discord, Telegram).

**Цвета квадратов:**

|Цвет|Статус|
|---|---|
|Синий (#1DA1F2)|Twitter подключен и активен|
|Белый (#FFFFFF)|GitHub подключен и активен|
|Фиолетовый (#5865F2)|Discord подключен и активен|
|Голубой (#0088CC)|Telegram подключен и активен|
|Полупрозрачный|Аккаунт подключен, но неактивен (status ≠ 'ok')|
|Серый прозрачный|Соцсеть не подключена|

**Hover** — показывает tooltip:
- Номер аккаунта
- Название соцсети
- Логин
- Статус (ok / error / not connected)

**Click** — копирует данные аккаунта в буфер обмена.

## Daily Projects Section

### Summary Cards

|Карточка|Описание|
|---|---|
|**TOTAL ATTEMPTS**|Общее количество попыток выполнения по всем проектам|
|**DONE**|Успешно выполненных задач / процент успеха|
|**FAILED**|Провалившихся задач|

### Projects HeatMap

Каждый проект отображается как строка с:

#### Project Card (слева)

- Название проекта
- Progress bar (зеленый = успех, красный = ошибки)
- Статистика:
  - ✔️ Successful: количество
  - Min|Max|Avg: время выполнения успешных задач (сек)
  - ❌ Failed: количество
  - Min|Max|Avg: время выполнения провалившихся задач (сек)
  - [✔️/❌] Rate: процент успеха

#### Heatmap Cells (справа)

Ячейки для каждого аккаунта (по maxAccountIndex).

**Цвета ячеек:**

|Цвет|Статус|Давность|
|---|---|---|
|Яркий зеленый|Успех|Сегодня|
|Темно-зеленый (#1a5c28)|Успех|Вчера|
|Очень темный зеленый (#123819)|Успех|2 дня назад|
|Почти черный зеленый (#0a1a0d)|Успех|3+ дня назад|
|Яркий красный|Ошибка|Сегодня|
|Темно-красный (#a32a27)|Ошибка|Вчера|
|Очень темный красный (#6b1e1d)|Ошибка|2 дня назад|
|Почти черный красный (#3a1210)|Ошибка|3+ дня назад|
|Прозрачный|Не выполнялось|—|

**Hover** — показывает tooltip:
- Номер аккаунта
- Название проекта
- Timestamp выполнения
- Время выполнения (сек)
- Статус (success / error)
- Текст отчета (если есть)

**Click** — копирует данные в буфер обмена.

### Сортировка проектов

URL-параметр `?sort=`:

|Значение|Описание|
|---|---|
|`lastActivity` (default)|По дате последней активности (новые сверху)|
|`name`|По имени проекта (алфавит)|
|`rate`|По проценту успеха (высокий → низкий)|

Пример: `/report.html?sort=rate`

### Idle Projects

Проекты без данных отображаются отдельной секцией **💤 Idle Projects** внизу страницы в виде карточек.

## Process Monitor Sidebar

Правая панель с мониторингом процессов по машинам.

Для каждой машины:

- **🖥 {MachineName}** — заголовок
- Список процессов:
  - Имя процесса
  - RAM (память)
  - Uptime (время работы)

**Hover** — показывает tooltip с полной командной строкой процесса.

**Click** — копирует командную строку в буфер обмена с уведомлением.

## Legend

Внизу каждой секции — легенда с расшифровкой цветов.

**Social Networks:**
- Цветные квадраты для каждой соцсети
- Полупрозрачный красный = ошибка
- Серый = не подключено

**Daily Projects:**
- Зеленые оттенки = успех (от сегодня до 3+ дней)
- Красные оттенки = ошибка (от сегодня до 3+ дней)
- Прозрачный = не выполнялось

## Edge cases

**Heatmap пустой** — нет данных в БД. Проверь что проекты выполняются и записывают результаты.

**Tooltip не показывается** — убедись что ячейка содержит данные. Пустые ячейки (прозрачные) не имеют tooltip.

**Процессы не отображаются** — проверь что переменные `processData_*` загружены. Если нет — backend не вернул данные процессов.

**Копирование не работает** — требуется HTTPS или localhost для работы `navigator.clipboard`. На HTTP копирование может не работать.

**Автообновление не работает** — проверь консоль браузера (F12). Возможно ошибка при загрузке данных.

**Idle Projects не показываются** — секция создается динамически только если есть проекты без данных. Если все проекты активны — секция не появится.
