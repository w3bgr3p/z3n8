

AI-анализ проектов через LLM. Генерирует текстовые отчеты по данным выполнения задач с использованием языковых моделей.

**Путь:** `/ai.html`  
**Хоткей:** `Alt+9`

---

## Обзор

AI Report — инструмент для автоматического анализа данных проектов с помощью LLM. Позволяет получить текстовый отчет о состоянии проекта, выявить проблемы и получить рекомендации.

---

## Интерфейс

### Header

**AI Report** — заголовок страницы.

**per-project LLM analysis** — подзаголовок.

### Controls

|Элемент|Описание|
|---|---|
|**Project**|Выпадающий список проектов из БД. Опция `— All Projects —` для сводного анализа|
|**Model**|Выбор LLM модели (загружается из `/ai-report/api/models`)|
|**Analyze**|Запустить анализ выбранного проекта|
|**Summary**|Запустить сводный анализ всех проектов|
|**Status label**|Счетчик при массовом анализе (например `3/10`)|

---

## Использование

### Анализ одного проекта

1. Выбери проект из списка **Project**
2. Выбери модель из списка **Model**
3. Нажми **Analyze**
4. Дождись завершения (появится карточка с результатом)

### Анализ всех проектов

1. Выбери `— All Projects —` в списке **Project**
2. Нажми **Analyze**
3. Проекты анализируются последовательно
4. Счетчик показывает прогресс (например `5/12`)

### Сводный анализ

Кнопка **Summary** запускает анализ всех проектов одним запросом:

- Отправляет агрегированные данные всех проектов в LLM
- Генерирует общий отчет о состоянии портфеля проектов
- Результат отображается в карточке `All Projects — Summary`

---

## Result Cards

Каждый результат отображается как карточка:

### Header

- **Название проекта** (или `All Projects — Summary`)
- **Статус badge**:
  - `analyzing…` (синий) — анализ в процессе
  - `done` (зеленый) — успешно завершен
  - `error` (красный) — ошибка
  - `cached` (серый) — загружен из кэша
- **Модель** — короткое имя модели (например `gpt-4`)
- **Timestamp** — дата и время анализа

### Body

Текст анализа от LLM.

При ошибке — текст ошибки красным цветом.

### Кнопка Clear

Появляется для кэшированных результатов. Удаляет результат из кэша и убирает карточку.

---

## Кэширование

Результаты анализа кэшируются на сервере:

- При открытии страницы загружаются из `/ai-report/api/cache`
- При повторном анализе того же проекта — используется кэш
- Кнопка **✕ clear** удаляет кэш через `DELETE /ai-report/api/cache?project={name}`

Кэш привязан к проекту, не к модели. При смене модели и повторном анализе — кэш перезаписывается.

## Backend API

### `/ai-report/api/models`

Возвращает список доступных LLM моделей:

```json
{
  "models": [
    "openai/gpt-4",
    "anthropic/claude-3-opus",
    "google/gemini-pro"
  ]
}
```

### `/ai-report/api/analyze`

POST-запрос для анализа одного проекта:

```json
{
  "project": "project_name",
  "model": "openai/gpt-4"
}
```

Ответ:

```json
{
  "analysis": "Текст анализа от LLM...",
  "model": "openai/gpt-4"
}
```

### `/ai-report/api/analyze-all`

POST-запрос для сводного анализа:

```json
{
  "model": "openai/gpt-4"
}
```

Ответ аналогичен `/analyze`.

### `/ai-report/api/cache`

GET — получить все кэшированные результаты:

```json
{
  "entries": [
    {
      "project": "project1",
      "analysis": "...",
      "model": "openai/gpt-4",
      "ts": "2026-04-04T21:00:00"
    }
  ]
}
```

DELETE — удалить кэш проекта:

```
DELETE /ai-report/api/cache?project=project1
```

## Edge cases

**"No projects"** — БД не содержит проектов. Проверь что [[Report]] генерирует данные.

**"Error loading" в списке моделей** — `/ai-report/api/models` недоступен или вернул ошибку. Проверь что LLM-сервис настроен.

**"Parse error" в списке моделей** — backend вернул невалидный JSON. Проверь логи сервера.

**Анализ зависает** — LLM-запрос может занимать 10-60 секунд в зависимости от модели и объема данных. Дождись завершения.

**Ошибка "HTTP 500"** — проблема на стороне сервера или LLM API. Проверь логи z3nIO и доступность LLM-сервиса.

**Кэш не очищается** — убедись что DELETE-запрос успешен. Проверь консоль браузера (F12).

**Карточки дублируются** — при повторном анализе старая карточка обновляется, а не создается новая. Если дублируются — возможно баг в коде (проверь `cardId` генерацию).
