

Мониторинг системы. Захват и анализ снимков состояния: процессы, сетевые подключения, диски, память, CPU. AI-аудит конфигурации.

**Путь:** `/system.html`  
**Хоткей:** `Alt+S`

---

## Обзор

System Snapshot — инструмент для сбора и анализа системной информации. Захватывает данные через PowerShell-скрипт, парсит их в структурированный вид и предоставляет интерактивный интерфейс для просмотра. Поддерживает AI-аудит для выявления проблем безопасности и производительности.

|Поле|Описание|
|---|---|
|**Host**|Имя машины|
|**RAM Used**|Использование памяти (GB и %)|

|**TCP Conns**|Количество установленных TCP-соединений|

**▶ Capture Now** — запустить захват снимка прямо сейчас.

## Section Navigation

## Content Area

Правая панель с содержимым выбранной секции.

### Типы секций

#### Key-Value (System Information, Memory, CPU)

Отображается как двухколоночная таблица:

- Левая колонка — название параметра
- Правая колонка — значение

Пример:
```
Hostname        : DESKTOP-1
OS              : Windows 11 Pro
Uptime          : 5 days 12:30:15
```

#### Table (Processes, Connections, Disks)

Интерактивная таблица с:

- **Сортировка** — клик по заголовку колонки (↑ / ↓)
- **Фильтр** — поле ввода над таблицей для фильтрации строк
- **Счетчик** — показывает количество отфильтрованных / всего строк
- **Цветовая индикация**:
  - Зеленый — статус `Running`, состояние `Established`
  - Синий (accent) — состояние `Listen`
  - Красный — использование > 85%
  - Желтый — использование 60-85%

**Inline bar** — для колонок с памятью (MEM_MB, TOTAL_MEM_MB) отображается горизонтальная полоска пропорционально значению.

#### Raw Text

Секции без структуры отображаются как моноширинный текст.

## Capture Now

Кнопка **▶ Capture Now** запускает:

1. POST-запрос на `/system-snapshot/capture`
2. Сервер выполняет PowerShell-скрипт для сбора данных
3. Возвращает raw-текст снимка
4. Снимок автоматически сохраняется в БД
5. Отображается в интерфейсе

Статус отображается в toolbar (например `Capturing…` → `Done`).

---

## Load from File

Для загрузки снимка из файла:

1. Нажми **From File (.txt)** — откроется диалог выбора файла
2. Или введи путь в поле **Last output path** и нажми **Load**

Поддерживаются файлы `.txt` и `.log` с выводом PowerShell-скрипта.

## Saved Snapshots Sidebar

Правая боковая панель с сохраненными снимками.

### Список снимков

Каждый снимок отображается как карточка:

- **Timestamp** — дата и время
- **Hostname** — имя машины
- **✕** — удалить снимок

Клик по карточке — загружает снимок.

Активный снимок подсвечен синей рамкой.

### AI Audit

AI-анализ текущего снимка через LLM.

**Model** — выбор модели из списка.

**Audit** — запустить анализ. Отправляет raw-текст снимка в `/system-snapshot/ai-audit`.

**Clear** — очистить кэшированный результат.

Результат показывает:

- Текст аудита (проблемы безопасности, рекомендации по оптимизации)
- Использованная модель
- Timestamp

Кэшируется на сервере. При повторном открытии снимка загружается из `/system-snapshot/ai-cache`.

## Backend API

### `/system-snapshot/capture`

POST-запрос для захвата снимка:

Ответ:

```json
{
  "raw": "## System Information\nHostname : ...\n..."
}
```

### `/system-snapshot/read-file`

POST-запрос для чтения файла:

```json
{
  "path": "C:\\path\\to\\snapshot.txt"
}
```

Ответ:

```json
{
  "raw": "..."
}
```

### `/system-snapshot/save`

POST-запрос для сохранения снимка в БД:

```json
{
  "raw": "..."
}
```

Ответ:

```json
{
  "id": 123
}
```

### `/system-snapshot/list`

GET — список сохраненных снимков:

```json
{
  "snapshots": [
    {
      "id": 123,
      "ts": "2026-04-04 21:00:00",
      "host": "DESKTOP-1"
    }
  ]
}
```

### `/system-snapshot/get`

GET — загрузить снимок по ID:

```
/system-snapshot/get?id=123
```

Ответ:

```json
{
  "raw": "...",
  "ts": "2026-04-04 21:00:00"
}
```

### `/system-snapshot/delete`

DELETE — удалить снимок:

```
DELETE /system-snapshot/delete?id=123
```

### `/system-snapshot/ai-audit`

POST-запрос для AI-аудита:

```json
{
  "model": "openai/gpt-4",
  "raw": "..."
}
```

Ответ:

```json
{
  "analysis": "Текст аудита...",
  "model": "openai/gpt-4",
  "ts": "2026-04-04T21:00:00"
}
```

### `/system-snapshot/ai-cache`

GET — получить кэшированный аудит.

DELETE — очистить кэш.

## Парсинг снимка

Снимок парсится автоматически:

1. Разделяется на секции по маркеру `## Section Name`
2. Каждая секция анализируется:
   - **Key-Value** — строки вида `Key  :  Value`
   - **Table** — первая строка = заголовки колонок, остальные = данные
   - **Raw** — если не подходит под KV или Table

3. Таблицы поддерживают:
   - Сортировку по любой колонке
   - Фильтрацию по содержимому
   - Цветовую индикацию

## Edge cases

**Capture не работает** — проверь что PowerShell-скрипт доступен на сервере и имеет права на выполнение.

**Файл не загружается** — убедись что путь корректен и файл существует. Проверь права доступа.

**Секция пустая** — возможно формат снимка не распознан. Проверь что файл содержит маркеры `## Section Name`.

**Таблица не сортируется** — убедись что колонка содержит данные. Пустые колонки могут не сортироваться корректно.

**AI Audit не работает** — проверь что модель выбрана и `/ai-report/api/models` возвращает список. Если модель недоступна — выберите другую.

**Снимок не сохраняется** — проверь подключение к БД в [[Config]].

**Глобальный фильтр не применяется** — фильтр работает только для таблиц. Key-Value секции не фильтруются.
